# Prompt: นำสไตล์ "LoveSave" มาปรับใช้กับ SOLARX

## ข้อควรคิดก่อน (สำคัญกว่าตัวโค้ด)

SOLARX คือแดชบอร์ดมอนิเตอร์อุปกรณ์ MPPT จริง มีแท็บดีบัก dp code, ตั้งค่าฮาร์ดแวร์, เงื่อนไขแจ้งเตือนความปลอดภัย (แรงดัน/อุณหภูมิเกิน) — เอาสไตล์การ์ตูนน่ารักแบบ LoveSave มาแทนที่**ทั้งแอป**จะขัดกับสิ่งที่คุยกันไว้ก่อนหน้า (Calm/Flat UI เหมาะกับแดชบอร์กข้อมูลจริงจังมากกว่า) และอาจทำให้ตัวเลขทางเทคนิคดูไม่น่าเชื่อถือ

**คำแนะนำ: หยิบมาแค่บางองค์ประกอบ ไม่ใช่ทั้งธีม** — เอาความอบอุ่น/มาสคอตมาใช้เฉพาะจุดที่เป็น "แรงจูงใจ" ส่วนหน้าข้อมูลจริง (กราฟ, ดีบัก, ตั้งค่า) ให้คงสไตล์เรียบมืออาชีพไว้เหมือนเดิม

## Prompt สำหรับ AI (คัดลอกวางได้เลย)

### บริบท
เพิ่มฟีเจอร์ "โซลาร์บัดดี้" ในแอป SOLARX (Next.js/TanStack Start + Tailwind) ซึ่งเป็นแดชบอร์ดมอนิเตอร์ MPPT Solar Charge Controller ที่มีอยู่แล้ว **ไม่ต้องเปลี่ยนดีไซน์หน้า Dashboard/History/Debug/Settings เดิม** — ฟีเจอร์นี้เป็นเลเยอร์เสริมแยกต่างหาก

### สิ่งที่ต้องการ

**1. มาสคอต "โซลาร์บัดดี้"**
- ตัวการ์ตูนชิบิ 1 ตัว (แนะนำ: เมล็ดพันธุ์/ต้นอ่อนใส่หมวกโซลาร์เซลล์ หรือดวงอาทิตย์หน้ายิ้ม) ปรากฏเป็น floating widget มุมล่างขวาของหน้า Dashboard เท่านั้น (ไม่ปนกับหน้าดีบัก/ตั้งค่า)
- แสดงอารมณ์ตามสถานะระบบจริง ผูกกับข้อมูลที่มีอยู่แล้ว:
  - PV power > 0 และแบตชาร์จอยู่ → ยิ้มสดใส
  - แบตต่ำ (SOC < 20%) → หน้ากังวล
  - เข้าเงื่อนไข alert (แรงดัน/อุณหภูมิเกิน) → หน้าตกใจ พร้อม tooltip บอกสาเหตุสั้น ๆ
  - กลางคืน/ไม่มีแดด → หลับ (ตากรอบ), โชว์ข้อความ "พรุ่งนี้เจอกันนะ"

**2. "ใบเสร็จพลังงานรายเดือน" (แรงบันดาลใจจากหน้าใบเสร็จ LoveSave)**
- สรุปท้ายเดือนแบบจำลองใบเสร็จจริง: เลขที่รายงาน, วันที่ออกรายงาน, breakdown เป็น % ของ (พลังงานที่ผลิตได้ / พลังงานที่ใช้ / พลังงานที่ประหยัดได้เทียบกับซื้อไฟจากการไฟฟ้า)
- ใช้ข้อมูลจาก `powerHistoryQuery("month")` ที่มีอยู่แล้วในโปรเจกต์
- แปลง kWh เป็นมูลค่าบาทโดยประมาณ (ให้ตั้งค่าราคาไฟต่อหน่วยได้ในหน้า "ตั้งค่า → แอป")

**3. Progress ring เป้าหมายการผลิตไฟ (ไม่ใช่การแข่งกับใคร)**
- ต่างจาก LoveSave ที่แข่ง "เธอ VS ฉัน" — สำหรับ SOLARX ให้เทียบ "เดือนนี้ VS เดือนที่แล้ว" แทน (เพราะเป็นระบบเดี่ยว ไม่ใช่คู่รัก)
- วงกลม progress แสดง % ของเป้าหมายการผลิตไฟที่ตั้งไว้ต่อเดือน

**4. โทนสี**
- คงพาเลตต์เดิมของ SOLARX (Solar Glow Dark theme: พื้นเข้ม, accent ส้ม/เขียว) — ไม่ใช้เขียวพาสเทลแบบ LoveSave เพราะจะขัดกับธีมมืดที่มีอยู่
- มาสคอตออกแบบให้เข้ากับ dark theme (ใช้ outline สว่างตัดกับพื้นเข้ม แทนสีพาสเทลอ่อน)

### ข้อจำกัด
- ห้ามให้มาสคอต/ใบเสร็จ/progress ring ไปปรากฏในหน้าดีบัก (`/debug`) หรือหน้าตั้งค่าอุปกรณ์ — ฟีเจอร์นี้อยู่เฉพาะ Dashboard และหน้าประวัติเท่านั้น
- ต้องมีสวิตช์ปิดมาสคอตได้ในหน้าตั้งค่า → แอป (ผู้ใช้บางคนอาจอยากได้แดชบอร์ดล้วน ๆ ไม่เอาการ์ตูน)
- ข้อมูลอารมณ์มาสคอตต้องคำนวณจาก state ที่มีอยู่แล้ว (`deviceStatusQueryWithInterval`, `evaluateAlerts`) ห้ามสร้างแหล่งข้อมูลใหม่ซ้ำซ้อน

## สรุป
เอาจุดแข็งของ LoveSave มาแค่ **"ความรู้สึกมีเพื่อนคู่คิด" กับ "การสรุปผลแบบจับต้องได้ (ใบเสร็จ)"** สองอย่างนี้ใช้ได้กับแอปข้อมูลจริงจังโดยไม่ทำลายความน่าเชื่อถือ ส่วนองค์ประกอบที่เป็น "การแข่งขันระหว่างคน" หรือ "โทนสีพาสเทลทั้งแอป" ไม่เหมาะกับ SOLARX เพราะเป็นระบบเดี่ยวสำหรับวิศวกร ไม่ใช่แอปโซเชียลระหว่างคู่รัก
